Introdução ao Liquid

Entenda o que é Shopify, o que é a linguagem Liquid e como ela transforma os dados da sua loja em páginas dinâmicas e personalizadas, tornando sua vitrine online única e funcional.

O que é Shopify?

É uma plataforma de comércio eletrônico completa. Permite que qualquer pessoa crie uma loja online para vender produtos (físicos ou digitais) ou serviços. Oferece ferramentas para gerenciar produtos, estoque, pagamentos, frete e marketing. Você não precisa se preocupar com servidores ou infraestrutura complexa.

Em resumo: A Shopify cuida da tecnologia para você focar em vender!

O que é Liquid?

Liquid é uma linguagem de template de código aberto criada pela Shopify e escrita em Ruby. Pense nela como uma ponte entre os dados da sua loja (produtos, coleções, informações do cliente) e o HTML que forma a aparência do seu site. Ela é usada para carregar conteúdo dinâmico nas páginas da sua loja.

Importante: Liquid não é uma linguagem de programação completa como Python ou JavaScript, mas sim uma linguagem focada em lógica de apresentação.

Para que serve o Liquid dentro dos temas Shopify?

  • Personalizar a aparência: Você usa Liquid para controlar como as informações da sua loja são exibidas. Por exemplo, mostrar o nome e o preço de um produto.
  • Criar páginas dinâmicas: Em vez de criar uma página HTML para cada produto, você cria um modelo (template) com Liquid. A Shopify usa esse modelo para gerar a página de cada produto automaticamente, preenchendo com os dados específicos daquele produto.
  • Adicionar lógica: Mostrar mensagens diferentes se um produto está em estoque ou não, listar produtos de uma coleção específica, exibir promoções, etc.

Exemplo simples: Imagine que você quer mostrar o nome da sua loja no rodapé de todas as páginas. Com Liquid, você faria algo assim no código do seu tema:

Code
<p>Rodapé da Loja: {{ shop.name }}</p>
→ Output (se o nome da sua loja for "Minha Loja Online")

Rodapé da Loja: Minha Loja Online

Explicação: shop.name é uma variável Liquid que contém o nome da sua loja, e {{ }} diz ao Liquid para mostrar esse valor.

OS BLOCOS DE CONSTRUÇÃO

Fundamentos do Liquid

Aprenda a sintaxe essencial, como usar variáveis para armazenar informações, como transformar dados com filtros e como acessar os dados da sua loja com objetos e seus atributos.

Sintaxe Básica: As Chaves Mágicas!

O Liquid usa dois tipos principais de delimitadores (as "chaves mágicas"):

  1. Chaves Duplas {{ }} - Para Saída (Output): Usadas para mostrar o valor de variáveis, objetos ou o resultado de filtros. Tudo o que está dentro de {{ }} será processado e o resultado final será exibido na página.
  2. Chave e Porcentagem {% %} - Para Lógica (Tags): Usadas para executar comandos lógicos, como condicionais (if/else), loops (for), atribuição de variáveis (assign), inclusão de outros arquivos (render), etc. Geralmente não imprimem nada diretamente, mas controlam o que será impresso.
Saída (Output) `{{ }}`
Olá, {{ customer.first_name }}!
O preço é {{ product.price | money }}.
→ Output (exemplo)

Olá, João!
O preço é R$ 19,99.

Lógica (Tags) `{% %}`
{% assign saudacao = "Bem-vindo" %}
{% if customer %}
  <p>{{ saudacao }}, {{ customer.first_name }}!</p>
{% else %}
  <p>{{ saudacao }} à nossa loja!</p>
{% endif %}
→ Output (se cliente "Maria" logada)

Bem-vindo, Maria!

→ Output (se cliente não logado)

Bem-vindo à nossa loja!

Variáveis: Guardando Informações

Variáveis são como "caixinhas" onde você pode guardar informações (texto, números, listas) para usar depois. No Liquid, você cria variáveis usando a tag {% assign %}.

Sintaxe: {% assign nome_da_variavel = valor %}

Exemplo:

Variáveis
{% assign nome_da_loja = "A Loja da Esquina" %}
{% assign ano_atual = "now" | date: "%Y" %}
{% assign itens_no_carrinho = cart.item_count %}

<h1>Bem-vindo à {{ nome_da_loja }}!</h1>
<p>Estamos em {{ ano_atual }}.</p>
<p>Você tem {{ itens_no_carrinho }} itens no carrinho.</p>
→ Output (Exemplo, se houver 2 itens no carrinho e ano for 2025)

Bem-vindo à A Loja da Esquina!

Estamos em 2025.

Você tem 2 itens no carrinho.

Explicação: nome_da_loja guarda o texto "A Loja da Esquina". ano_atual guarda o ano atual (usamos o filtro date para formatar a data). itens_no_carrinho guarda o número de itens que o cliente tem no carrinho (vindo do objeto cart). Depois, usamos {{ }} para mostrar os valores dessas variáveis.

Filtros: Transformando Dados

Filtros são usados para modificar a saída de números, strings (textos), variáveis e objetos. Eles são colocados dentro de {{ }} ou {% %} e são separados do valor por um pipe |. Você pode encadear múltiplos filtros.

Sintaxe: {{ valor | filtro1 | filtro2: argumento }}

Exemplos Comuns:

Filtros
{% assign nome_produto = "camiseta básica" %}
{% assign preco = 29.9 %}

<!-- Texto em maiúsculas -->
<p>{{ nome_produto | upcase }}</p>

<!-- Formatar como dinheiro -->
<p>Preço: {{ preco | money }}</p>

<!-- Pegar os primeiros 5 caracteres -->
<p>Código: {{ "ABCDEFG12345" | slice: 0, 5 }}</p>

<!-- Formatar data -->
<p>Data do pedido: {{ "2025-05-31" | date: "%d/%m/%Y" }}</p>

<!-- Pluralizar (se cart.item_count for 1, mostra 'item', senão 'itens') -->
<p>Você tem {{ cart.item_count }} {{ cart.item_count | pluralize: 'item', 'itens' }}</p>
→ Output (Exemplo, se cart.item_count for 1)

CAMISETA BÁSICA

Preço: R$ 29,90

Código: ABCDE

Data do pedido: 31/05/2025

Você tem 1 item

Explicação: upcase: Converte o texto para maiúsculas. money: Formata um número como valor monetário. slice: 0, 5: Pega uma "fatia" do texto. date: "%d/%m/%Y": Formata uma data. pluralize: 'item', 'itens': Escolhe a forma singular ou plural.

Operadores: Fazendo Comparações e Cálculos

Operadores são usados principalmente dentro de tags de lógica ({% %}), como if, elsif, unless e case.

  • Comparação: == (igual a), != (diferente de), < (menor que), > (maior que), <= (menor ou igual a), >= (maior ou igual a).
  • Lógicos: and (E lógico), or (OU lógico).
  • Contém: contains (Verifica se uma string contém uma substring, ou se um array contém um elemento).

Exemplos:

Operadores
{% assign idade = 20 %}
{% assign produto_em_estoque = true %}
{% assign tags_do_produto = "novo, promoção, verão" %}
{% assign preco = 29.9 %}

{% if idade >= 18 %}
  <p>Você é maior de idade.</p>
{% endif %}

{% if produto_em_estoque and preco < 50 %}
  <p>Produto disponível e barato!</p>
{% endif %}

{% if tags_do_produto contains "promoção" %}
  <p>Este produto está em promoção!</p>
{% endif %}

{% assign numero = 10 %}
{% if numero > 5 and numero < 15 %}
  <p>O número está entre 5 e 15.</p>
{% endif %}
→ Output (considerando preco como 29.9)

Você é maior de idade.

Produto disponível e barato!

Este produto está em promoção!

O número está entre 5 e 15.

Objetos e Atributos

Objetos: A Base de Tudo na Shopify
Objetos no Liquid são como grandes "contêineres" que agrupam dados relacionados. Pense neles como fichas de informação. A Shopify disponibiliza vários objetos globais que você pode acessar em qualquer lugar do seu tema. Eles contêm informações sobre sua loja, produtos, coleções, carrinho, cliente, etc.

Principais Objetos Globais incluem: product, collection, cart, customer, shop, settings, pages, article, blog, linklists. Você acessa os dados dentro de um objeto usando um ponto (.).

Atributos: Acessando Dados dos Objetos
Atributos são as informações específicas guardadas dentro de um objeto. Você usa a notação de ponto (.) para acessar um atributo de um objeto.

Sintaxe: {{ objeto.atributo }} ou {{ objeto.outro_objeto.atributo_profundo }}

Exemplo com o objeto `product`: Se você está na página de um produto chamado "Camiseta Estilosa" que custa R$ 59,90:

Acessando Atributos do Produto
<h1>{{ product.title }}</h1>
<p>Descrição: {{ product.description }}</p>
<p>Preço: {{ product.price | money }}</p>
<p>Vendedor: {{ product.vendor }}</p>
<p>Tipo: {{ product.type }}</p>
<img src="{{ product.featured_image | image_url: width: 300 }}" alt="{{ product.featured_image.alt | escape }}">
→ Output (Exemplo)

Camiseta Estilosa

Descrição: Esta camiseta é feita de algodão 100% orgânico e tem uma estampa exclusiva.

Preço: R$ 59,90

Vendedor: Marca Legal

Tipo: Vestuário

Explicação: product.title: Acessa o título. product.description: Acessa a descrição. product.price | money: Acessa e formata o preço. product.vendor: Acessa o fornecedor. product.type: Acessa o tipo. product.featured_image | image_url: width: 300: Pega a URL da imagem principal e a redimensiona.

TOMANDO DECISÕES E REPETINDO AÇÕES

Controle de Fluxo

Domine como controlar o que é exibido com condicionais (if, unless, case) e como repetir blocos de HTML com loops (for) para criar páginas verdadeiramente dinâmicas e responsivas aos dados.

Condicionais: `if`, `else`, `elsif`

Condicionais permitem que seu código execute diferentes blocos dependendo se uma condição é verdadeira ou falsa. {% if condicao %} executa o bloco se a condição for verdadeira. {% elsif outra_condicao %} (opcional) testa outra condição se a primeira for falsa. {% else %} (opcional) executa se todas as condições anteriores forem falsas. Sempre feche com {% endif %}.

Exemplo de `if/else` (Disponibilidade do Produto)
{% if product.available %}
  <button type="submit" name="add" class="btn btn-primary">Adicionar ao Carrinho</button>
{% else %}
  <p class="text-red-600">Produto Esgotado</p>
{% endif %}
→ Output (se product.available for false)

Produto Esgotado

Exemplo com `elsif`
{% assign pontuacao = 75 %}
{% if pontuacao >= 90 %}
  <p>Nota: A</p>
{% elsif pontuacao >= 70 %}
  <p>Nota: B</p>
{% elsif pontuacao >= 50 %}
  <p>Nota: C</p>
{% else %}
  <p>Nota: D - Precisa melhorar!</p>
{% endif %}
→ Output

Nota: B

Condicionais: `unless` (A Menos Que)

unless é o oposto de if. {% unless condicao %} executa o bloco se a condição for falsa. Pode ter um {% else %} e deve ser fechado com {% endunless %}.

Exemplo de `unless` (Carrinho não vazio)
{% unless cart.empty? %}
  <p>Você tem itens no seu carrinho!</p>
  <a href="/cart">Ver Carrinho</a>
{% else %}
  <p>Seu carrinho está vazio. Que tal adicionar alguns produtos?</p>
{% endunless %}
→ Output (se carrinho tiver itens, cart.empty? é false)

Você tem itens no seu carrinho!

Ver Carrinho

Você poderia escrever o mesmo com if not cart.empty? ou if cart.item_count > 0. unless é apenas uma forma mais legível em certas situações.

Condicionais: `case` (Múltiplas Escolhas)

A tag case é útil quando você tem uma variável e quer executar diferentes blocos de código dependendo do valor dessa variável. É como uma série de if/elsif mais organizada para comparações de igualdade. Use {% case variavel %}, {% when valor %}`...`{% endcase %}.

Exemplo de `case` (Tipo de Produto)
{% assign tipo_produto = product.type %}
{% case tipo_produto %}
  {% when "Camiseta" %}
    <p>Dica: Lave esta camiseta à mão para maior durabilidade.</p>
  {% when "Caneca" %}
    <p>Esta caneca pode ir ao micro-ondas!</p>
  {% when "Livro" or "Revista" %}
    <p>Boa leitura!</p>
  {% else %}
    <p>Confira as especificações do produto para cuidados especiais.</p>
{% endcase %}
→ Output (se product.type for "Caneca")

Esta caneca pode ir ao micro-ondas!

Loops: `for`

Loops for são usados para iterar (passar por cada item) sobre uma coleção de itens, como os produtos de uma coleção, as imagens de um produto, ou os itens no carrinho. {% for item in colecao %}...{% endfor %}.

Listando Produtos de uma Coleção com `for`
<h2>Produtos da Coleção: {{ collection.title }}</h2>
<ul>
  {% for product_item in collection.products %}
    <li>
      <a href="{{ product_item.url }}">
        {{ product_item.title }} - {{ product_item.price | money }}
      </a>
    </li>
  {% endfor %}
</ul>
→ Output (Exemplo para uma coleção com 2 produtos)

Produtos da Coleção: Roupas de Verão

  • Camiseta Praia - R$ 49,90
  • Bermuda Surf - R$ 79,90

Explicação: O loop passa por cada produto em collection.products. Em cada passagem, product_item se refere ao produto atual, permitindo acessar product_item.title, product_item.price, etc.

Ajudantes e Parâmetros do Loop `for`

Dentro de um loop for, você tem acesso ao objeto forloop com informações como forloop.index (índice atual, começa em 1), forloop.first (true na primeira iteração), forloop.last (true na última). Parâmetros como limit:<numero>, offset:<numero> e reversed podem modificar o comportamento do loop. Um bloco {% else %} dentro do for é executado se a coleção estiver vazia.

Exemplo com Ajudantes e Parâmetros do `for`
<h3>Nossos 2 principais produtos:</h3>
<ul>
  {% for product in collection.products limit:2 %}
    <li>
      {{ forloop.index }}. {{ product.title }}
      {% if forloop.first %} (Primeiro da lista!){% endif %}
      {% if forloop.last %} (Último desta seleção!){% endif %}
    </li>
  {% else %}
    <li>Nenhum produto encontrado nesta coleção.</li>
  {% endfor %}
</ul>
→ Output (se a coleção tiver produtos)

Nossos 2 principais produtos:

  • 1. Camiseta Praia (Primeiro da lista!)
  • 2. Bermuda Surf (Último desta seleção!)

Loops: `cycle` (Alternando Valores)

A tag cycle é usada dentro de um loop for para alternar entre uma série de valores a cada iteração. Útil para aplicar classes CSS alternadas (ex: "zebra striping"). Sintaxe: {% cycle 'valor1', 'valor2' %}. Pode-se nomear grupos de ciclo: {% cycle 'grupo': 'a', 'b' %}.

Exemplo de `cycle` para Classes CSS
<div class="lista-produtos">
  {% for product in collection.products %}
    <div class="produto-item {% cycle 'par', 'impar' %}">
      <h4>{{ product.title }}</h4>
    </div>
  {% endfor %}
</div>
<style>
  .par { background-color: #f0f0f0; }
  .impar { background-color: #ffffff; }
  .produto-item { padding: 10px; }
</style>
→ Output (HTML para 3 produtos)

Camiseta Praia

Bermuda Surf

Chapéu Palha

Loops: `break` e `continue` (Controle de Fluxo)

Dentro de um loop for: {% break %} interrompe o loop imediatamente. {% continue %} pula para a próxima iteração do loop.

Exemplo com `break`
<h4>Primeiro produto abaixo de R$50,00:</h4>
{% for product in collection.products %}
  {% if product.price < 5000 %} <!-- Preço em centavos -->
    <p>{{ product.title }} - {{ product.price | money }}</p>
    {% break %} <!-- Encontrou, então para o loop -->
  {% endif %}
{% endfor %}
→ Output (se o segundo produto for "Chapéu Palha" a R$35,00)

Primeiro produto abaixo de R$50,00:

Chapéu Palha - R$ 35,00

Exemplo com `continue`
<h4>Produtos Disponíveis:</h4>
<ul>
  {% for product in collection.products %}
    {% unless product.available %}
      {% continue %} <!-- Pula para o próximo produto se este estiver indisponível -->
    {% endunless %}
    <li>{{ product.title }}</li>
  {% endfor %}
</ul>
→ Output (se "Bermuda Surf" estiver indisponível)

Produtos Disponíveis:

  • Camiseta Praia
  • Chapéu Palha
OS DADOS DA SUA LOJA AO SEU ALCANCE

Trabalhando com Objetos Shopify

A Shopify disponibiliza diversos objetos globais que contêm todas as informações sobre sua loja. Explore os mais comuns e veja exemplos práticos de como usá-los. Lembre-se que os atributos exatos podem variar ligeiramente entre temas ou com apps instalados, mas os principais são consistentes.

Visão Geral dos Atributos Comuns (detalhes e mais atributos dentro de cada objeto abaixo):

  • product: .title, .handle, .price, .compare_at_price, .description, .featured_image, .images, .variants, .tags, .available.
  • collection: .title, .description, .products (array), .image.
  • cart: .total_price, .item_count, .items (array), .empty?.
  • settings: Acessa configurações do Personalizador de Temas (ex: settings.cor_primaria).
  • pages: Acessa páginas estáticas pelo handle (ex: pages['sobre-nos'].title).
Objeto `product`

O objeto product é acessível em páginas de produto (product.liquid, snippets usados nela) e quando você itera sobre collection.products, cart.items, etc.

Exemplo: Mostrar detalhes de um produto

Detalhes do Produto
<article class="produto-detalhe">
  <h1>{{ product.title }}</h1>
  {% if product.vendor != blank %}
    <p class="vendor">Marca: {{ product.vendor }}</p>
  {% endif %}
  <div class="preco">
    <span class="preco-atual">{{ product.price | money }}</span>
    {% if product.compare_at_price > product.price %}
      <del class="preco-antigo">{{ product.compare_at_price | money }}</del>
      {% assign economia = product.compare_at_price | minus: product.price %}
      <span class="economia">Você economiza {{ economia | money }}!</span>
    {% endif %}
  </div>
  {% if product.featured_image %}
    <img src="{{ product.featured_image | image_url: width: 600 }}"
         alt="{{ product.featured_image.alt | escape | default: product.title | escape }}"
         class="imagem-destaque">
  {% endif %}
  <div class="descricao">
    {{ product.description }}
  </div>
  <h4>Tags:</h4>
  <ul>
    {% for tag in product.tags %}
      <li>{{ tag }}</li>
    {% else %}
      <li>Este produto não possui tags.</li>
    {% endfor %}
  </ul>
  {% if product.available %}
    <button>Comprar Agora</button>
  {% else %}
    <p>Produto Indisponível no momento.</p>
  {% endif %}
</article>
→ Output (Exemplo para um produto em promoção "Camiseta Premium")

Camiseta Premium

Marca: SuperGrife

R$ 79,90R$ 99,90Você economiza R$ 20,00!

Uma camiseta de alta qualidade, feita com o melhor algodão.

Tags:

  • algodao
  • premium
  • novidade

Explicação: Mostramos título, vendedor (se houver). Calculamos e mostramos a economia se houver compare_at_price. Exibimos a imagem principal, com um alt padrão caso o específico não exista. Listamos as tags ou uma mensagem se não houver tags. Mostramos o botão de compra ou mensagem de indisponibilidade.

Objeto `collection`

O objeto collection é acessível em páginas de coleção (collection.liquid) e quando você se refere a uma coleção específica (ex: collections['ofertas']).

Exemplo: Cabeçalho da coleção e grade de produtos com paginação

Coleção com Paginação
<!-- Em collection.liquid -->
<div class="cabecalho-colecao">
  <h1>{{ collection.title }}</h1>
  {% if collection.description != blank %}
    <div class="descricao-colecao">{{ collection.description }}</div>
  {% endif %}
  {% if collection.image %}
    <img src="{{ collection.image | image_url: width: 800 }}" alt="{{ collection.title | escape }}">
  {% endif %}
</div>

{% paginate collection.products by 8 %} <!-- Mostrar 8 produtos por página -->
  <div class="grid-produtos">
    {% for product_item in collection.products %}
      <div class="produto-card">
        <a href="{{ product_item.url | within: collection }}">
          {% if product_item.featured_image %}
            <img src="{{ product_item.featured_image | image_url: width: 250 }}" alt="{{ product_item.featured_image.alt | escape }}">
          {% else %}
            <div style="width:250px; height:250px; background:#eee; display:flex; align-items:center; justify-content:center;">Sem Imagem</div>
          {% endif %}
          <h3>{{ product_item.title }}</h3>
          <p>{{ product_item.price | money }}</p>
        </a>
      </div>
    {% else %}
      <p>Nenhum produto encontrado nesta coleção.</p>
    {% endfor %}
  </div>

  {% if paginate.pages > 1 %}
    <nav class="paginacao">
      {{ paginate | default_pagination }}
    </nav>
  {% endif %}
{% endpaginate %}
→ Output (Estrutura HTML gerada)

Um cabeçalho com título, descrição e imagem da coleção. Uma grade de produtos. Se houver mais de 8 produtos, links de paginação ("Anterior 1 2 3 Próxima") serão exibidos.

Explicação: {% paginate collection.products by 8 %} habilita a paginação. product_item.url | within: collection garante que o link do produto mantenha o contexto da coleção atual. {{ paginate | default_pagination }} renderiza os links de paginação padrão.

Objeto `cart`

O objeto cart está disponível em todas as páginas e contém informações sobre o carrinho de compras do usuário atual.

Exemplo: Resumo do carrinho (geralmente em cart.liquid ou um snippet)

Resumo do Carrinho
{% if cart.empty? %}
  <h2>Seu carrinho está vazio</h2>
  <p><a href="/collections/all">Continuar comprando</a></p>
{% else %}
  <h2>Seu Carrinho de Compras</h2>
  <form action="/cart" method="post" novalidate>
    <table>
      <thead>
        <tr><th>Produto</th><th>Preço</th><th>Quantidade</th><th>Total</th><th> </th></tr>
      </thead>
      <tbody>
        {% for item in cart.items %}
          <tr>
            <td>
              <a href="{{ item.url }}">
                <img src="{{ item.image | image_url: width: 100 }}" alt="{{ item.title | escape }}">
                {{ item.product.title }}
                {% unless item.variant.title contains 'Default' %}<br><small>{{ item.variant.title }}</small>{% endunless %}
              </a>
            </td>
            <td>{{ item.price | money }}</td>
            <td><input type="number" name="updates[]" id="updates_{{ item.key }}" value="{{ item.quantity }}" min="0"></td>
            <td>{{ item.line_price | money }}</td>
            <td><a href="/cart/change?line={{ forloop.index }}&quantity=0">Remover</a></td>
          </tr>
        {% endfor %}
      </tbody>
    </table>
    <div>
      <h3>Subtotal: {{ cart.total_price | money }}</h3>
      <p>Taxas e frete calculados no checkout.</p>
      <button type="submit" name="update">Atualizar Carrinho</button>
      <button type="submit" name="checkout">Finalizar Compra</button>
    </div>
  </form>
{% endif %}
→ Output (Estrutura HTML se o carrinho tiver itens)

Uma tabela listando cada item: imagem, título, variante, preço unitário, campo de quantidade, total da linha e link para remover. Um subtotal do carrinho. Botões para atualizar e finalizar.

Explicação: Iteramos sobre cart.items. name="updates[]" no input de quantidade é importante para a Shopify processar as atualizações. O link "Remover" usa quantity=0 para remover o item.

Objeto `customer`

O objeto customer contém informações sobre o cliente que está logado. Se ninguém estiver logado, customer será nil (nulo).

Exemplo: Saudação ao cliente e links da conta (geralmente no cabeçalho)

Saudação e Links da Conta
<div class="conta-usuario">
  {% if customer %}
    <p>
      Olá, {{ customer.first_name | default: "Cliente" }}!
      (<a href="{{ routes.account_logout_url }}">Sair</a>)
    </p>
    <a href="{{ routes.account_url }}">Minha Conta</a>
    {% if customer.orders.size > 0 %}
      | <a href="{{ routes.account_orders_url }}">Meus Pedidos</a>
    {% endif %}
  {% else %}
    <p>
      <a href="{{ routes.account_login_url }}">Entrar</a> ou
      <a href="{{ routes.account_register_url }}">Criar Conta</a>
    </p>
  {% endif %}
</div>
→ Output (Se cliente "Ana Silva" logada e com pedidos)

Olá, Ana! (Sair)

Minha Conta | Meus Pedidos

Nota: routes.account_url, routes.account_login_url etc., são a forma moderna e recomendada de gerar URLs de conta. customer.first_name | default: "Cliente" usa "Cliente" se o nome não estiver preenchido. customer.orders.size verifica se há pedidos.

Configurações do Tema

Objeto `pages`

O objeto pages permite acessar o conteúdo de páginas estáticas criadas no admin da Shopify (Loja Virtual > Páginas). Você acessa uma página específica usando seu "handle" (o identificador único na URL, ex: sobre-nos para a URL /pages/sobre-nos).

Exercício 3: Mostrar Preço Promocional

No arquivo product.liquid, modifique o código para mostrar o preço de venda normal (product.price). Apenas se houver um preço de comparação (product.compare_at_price) e ele for maior que o preço de venda, mostre o preço de comparação riscado (<del>) e uma mensagem de economia.

Ver Solução Comentada
Solução: Preço Promocional
<!-- No arquivo templates/product.liquid ou uma seção de produto -->

<div class="preco-produto">
  <span class="preco-venda">{{ product.price | money }}</span>

  {% if product.compare_at_price > product.price %}

    </span>
  {% endif %}
</div>

Desafio Iniciante: "Frete Grátis Acima de X"

Na página do carrinho, mostre uma mensagem diferente dependendo do valor total do carrinho (cart.total_price). Se o total for R$ 150,00 (15000 em centavos) ou mais, mostre: "Parabéns! Você ganhou frete grátis!". Caso contrário, mostre: "Adicione mais R$ YYY,YY ao carrinho para ganhar frete grátis!" (onde YYY,YY é o valor faltante).

Ver Solução Comentada
Solução: Desafio Frete Grátis
<!-- No template do carrinho (ex: cart.liquid) -->

{% assign limite_frete_gratis = 15000 %} <!-- R$ 150,00 em centavos -->
{% assign total_carrinho_centavos = cart.total_price %}

<div class="mensagem-frete p-4 my-4 rounded-md">
  {% if total_carrinho_centavos >= limite_frete_gratis %}
    <p style="color: green; font-weight: bold;">
      Parabéns! Você ganhou frete grátis!
    </p>
  {% else %}
    {% assign valor_faltante = limite_frete_gratis | minus: total_carrinho_centavos %}

      Adicione mais <strong style="color: red;">{{ valor_faltante | money }}</strong> 
      ao carrinho para ganhar frete grátis!
    </p>
  {% endif %}
</div>

Resultado Esperado: Se o carrinho tiver R$ 150,00 ou mais, a mensagem de frete grátis em verde aparecerá. Se tiver menos, por exemplo R$ 124,50, a mensagem será: "Adicione mais R$ 25,50 ao carrinho para ganhar frete grátis!" (com o valor faltante em vermelho/destacado).

PRÓXIMO NÍVEL DE DESENVOLVIMENTO

Tópicos Avançados em Liquid

Aprofunde seus conhecimentos em Liquid aprendendo a organizar seu código de forma eficiente, criar componentes reutilizáveis e construir temas modulares e performáticos com snippets e seções, seguindo as melhores práticas da Shopify.

Reutilizando Código: `include` (Legado) e `render` (Moderno)

Para evitar repetir o mesmo bloco de código Liquid em vários lugares, você pode criar "pedaços" de código reutilizáveis, chamados snippets.

  • {% include 'nome-do-snippet' %} (Legado): Inclui o conteúdo do arquivo snippets/nome-do-snippet.liquid. Importante: As variáveis dentro do snippet incluído compartilham o mesmo escopo do arquivo que o incluiu, podendo levar a efeitos colaterais. A passagem de variáveis é feita com with ou nomeando parâmetros.
  • {% render 'nome-do-snippet' %} (Moderno e Recomendado): Também inclui o snippet, mas cria um escopo de variáveis isolado. Variáveis devem ser passadas explicitamente (ex: {% render 'meu-snippet', variavel_para_snippet: valor %}). Isso torna o código mais modular, previsível e geralmente oferece melhor performance.

Exemplo com `render` (Criando um Card de Produto Reutilizável):

1. Crie o Snippet (snippets/card-produto.liquid):

snippets/card-produto.liquid
{% comment %} Arquivo: snippets/card-produto.liquid {% endcomment %}
<div class="card">
  <a href="{{ p.url | within: collection_context }}">
    {% if p.featured_image %}
      <img src="{{ p.featured_image | image_url: width: 300 }}" alt="{{ p.featured_image.alt | escape }}">
    {% else %}
      <div class="placeholder-imagem">{{ 'product-1' | placeholder_svg_tag: 'placeholder-svg' }}</div>
    {% endif %}
    <h4>{{ p.title }}</h4>
    <p class="preco">{{ p.price | money }}</p>
    {% if p.compare_at_price > p.price %}
      <p class="preco-comparacao"><del>{{ p.compare_at_price | money }}</del></p>
    {% endif %}
  </a>
</div>

2. Use o Snippet em uma Página de Coleção (collection.liquid):

templates/collection.liquid (trecho)
{% comment %} Arquivo: templates/collection.liquid {% endcomment %}
<div class="grid-produtos">
  {% for produto_da_iteracao in collection.products %}
    {% render 'card-produto', p: produto_da_iteracao, collection_context: collection %}
  {% endfor %}
</div>
→ Output

Uma grade de produtos, onde cada produto é exibido usando o estilo definido em card-produto.liquid.

Explicação: No collection.liquid, o loop for passa por cada produto_da_iteracao. {% render 'card-produto', p: produto_da_iteracao, collection_context: collection %} chama o snippet e passa o produto atual como p e a coleção como collection_context.

Seções: Blocos Modulares para o Personalizador

As Seções (Sections) são arquivos Liquid que definem blocos de conteúdo modulares e configuráveis que os lojistas podem adicionar, remover, reordenar e personalizar através do Editor de Temas da Shopify. Elas são a base dos temas modernos da Shopify (Online Store 2.0). Cada seção é um arquivo .liquid na pasta /sections e combina HTML/Liquid, um {% schema %} JSON para as configurações, e opcionalmente CSS e JavaScript.

Estrutura básica de um arquivo de seção (ex: sections/titulo-personalizado.liquid):

sections/titulo-personalizado.liquid
<!-- HTML e Liquid da Seção -->
<div class="container-titulo" style="background-color: {{ section.settings.cor_fundo }};">
  <h2 style="color: {{ section.settings.cor_texto_titulo }}; font-size: {{ section.settings.tamanho_fonte }}px;">
    {{ section.settings.texto_do_titulo | escape }}
  </h2>
</div>

{% stylesheet %}
  .container-titulo { padding: 20px; text-align: center; }
{% endstylesheet %}

{% schema %}
  {
    "name": "Título Personalizado",
    "settings": [
      {
        "type": "text",
        "id": "texto_do_titulo",
        "label": "Texto do Título",
        "default": "Meu Título Incrível"
      },
      { "type": "color", "id": "cor_texto_titulo", "label": "Cor do Texto", "default": "#333333" },
      { "type": "color", "id": "cor_fundo", "label": "Cor de Fundo", "default": "#FFFFFF" },
      { "type": "range", "id": "tamanho_fonte", "label": "Tamanho da Fonte (px)", "min": 16, "max": 72, "default": 32 }
    ],
    "presets": [{ "name": "Título Personalizado" }]
  }
{% endschema %}

No Editor de Temas, o lojista adiciona a seção "Título Personalizado" e customiza os campos. No Liquid, acessa-se com {{ section.settings.ID_DA_CONFIGURACAO }}.

Blocos Dinâmicos dentro de Seções

Além das settings (configurações) da seção, podem existir blocks (blocos). Blocos permitem que o lojista adicione, remova e reordene múltiplos itens *dentro* de uma seção (ex: slides, depoimentos). Cada tipo de bloco tem suas próprias configurações. No Liquid, itera-se sobre section.blocks.

Exemplo: Seção "Lista de Ícones com Texto" (sections/lista-icones.liquid)

sections/lista-icones.liquid (trecho do schema e loop)
<!-- HTML e Liquid da Seção -->
<div class="grid-icones">
  {% for block in section.blocks %}
    <div class="item-icone" {{ block.shopify_attributes }}>
      {% case block.type %}
        {% when 'icone_texto' %}
          {% if block.settings.imagem_icone %}
            <img src="{{ block.settings.imagem_icone | image_url: width: 80 }}">
          {% endif %}
          <h3>{{ block.settings.titulo_item_icone }}</h3>
          <div>{{ block.settings.texto_item_icone }}</div>
      {% endcase %}
    </div>
  {% endfor %}
</div>

{% schema %}
{
  "name": "Lista de Ícones com Texto",
  "settings": [ /* ... configurações da seção ... */ ],
  "blocks": [
    {
      "type": "icone_texto",
      "name": "Ícone com Texto",
      "limit": 4,
      "settings": [
        { "type": "image_picker", "id": "imagem_icone", "label": "Ícone" },
        { "type": "text", "id": "titulo_item_icone", "label": "Título" },
        { "type": "richtext", "id": "texto_item_icone", "label": "Texto" }
      ]
    }
  ],
  "presets": [{ "name": "Lista de Ícones", "blocks": [ { "type": "icone_texto" } ] }]
}
{% endschema %}

No Editor, o lojista adiciona blocos do tipo "Ícone com Texto" e preenche suas configurações. {{ block.shopify_attributes }} é essencial para o editor. Acessa-se as configurações do bloco com block.settings.ID_DA_CONFIGURACAO_DO_BLOCO.

Boas Práticas de Performance e Código Limpo

  • Minimize Loops e Consultas: Evite loops aninhados. Use limit em for. Recupere objetos específicos diretamente (all_products['handle']) em vez de iterar all_products (use com cautela).
  • Imagens Otimizadas: Sempre especifique tamanho (width, height, crop) com image_url ou image_tag para que a Shopify sirva imagens redimensionadas.
  • Fontes Eficientes: Use font_picker nas configurações e garanta font_display: swap; para carregamento rápido.
  • Comente Seu Código: Use {% comment %} ... {% endcomment %} para explicar lógicas complexas.
  • Snippets Focados: Crie snippets (render) pequenos e com responsabilidade única.
  • Escopo com `render`: Prefira render a include para melhor encapsulamento.
  • Evite Lógica Excessiva: Se a lógica for muito complexa, considere JavaScript (cliente) ou Shopify Apps (backend).
  • Use `routes` para URLs: Para URLs internos ({{ routes.cart_url }}), garantindo correção e localização.
  • Teste Cenários Diversos: Coleções vazias, produtos sem imagem, etc., usando {% else %} em loops e condicionais.
