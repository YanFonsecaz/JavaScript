Módulo 1: Fundamentos do Shopify (Iniciante)

Antes de colocar a mão na massa, é crucial entender alguns conceitos básicos da plataforma.

O que é Shopify?

Shopify é uma plataforma de comércio eletrônico completa que permite a qualquer pessoa criar uma loja online para vender seus produtos ou serviços...

Conceitos Essenciais:

  • Loja (Store): É a sua loja virtual completa...
  • Tema (Theme): É o "esqueleto" e o "visual" da sua loja online...
  • Produtos, Coleções, Clientes: Itens à venda...
  • Carrinho (Cart): Onde os clientes adicionam os produtos...
  • Checkout: O processo final de compra...
  • Apps (Aplicativos): Extensões que adicionam funcionalidades extras...
  • APIs (Interfaces de Programação de Aplicativos): Permitem que diferentes softwares "conversem"...
  • Liquid: A linguagem de template do Shopify...
  • Metafields (Campos Personalizados): Permitem adicionar informações extras...
Exercício Prático Módulo 1:
  1. Pense em um tipo de produto que você gostaria de vender (mesmo que fictício).
  2. Liste 3 informações específicas sobre esse produto que você gostaria de mostrar na página dele (além do nome e preço). Uma delas poderia ser um "metafield"?

Exemplo de Resposta:

Tipo de Produto: Camiseta de banda de rock.

Informações Adicionais:

  1. Nome da Banda (Texto simples).
  2. Ano de Lançamento do Álbum da Estampa (Número) - Isto poderia ser um metafield do tipo "Número Inteiro" ou "Texto".
  3. Material do Tecido (Texto, ex: "100% Algodão Penteado Fio 30.1").

Explicação: O "Ano de Lançamento" é um bom candidato para metafield porque é uma informação específica do produto que não tem um campo padrão no Shopify. O "Material do Tecido" também poderia ser um metafield se você quisesse estruturá-lo ou usá-lo para filtros avançados.

Você aprendeu o que é o Shopify e os blocos de construção fundamentais de qualquer loja na plataforma. Entender esses termos é o primeiro passo.

  </main>

  {% section 'footer' %} <!-- Exemplo de inclusão de seção de rodapé -->
</body>

Os itens 1, 2 e 3 são ações a serem realizadas na sua loja de desenvolvimento.

Módulo 3: Introdução ao Liquid (Iniciante)

Liquid é a espinha dorsal da personalização de temas Shopify.

O que é Liquid e por que usá-lo?

Liquid é uma linguagem de template de código aberto criada pelo Shopify...

Sintaxe Básica:

  • Objetos ({{ }}): Usados para exibir (output) dados...
  • ... (restante da sintaxe) ...

Exemplos Práticos:

1. Exibindo o nome da loja: ...

2. Listando produtos na página de coleção: ...

Exercício Prático Módulo 3:
  1. Em templates/product.liquid, encontre onde o título do produto é exibido.
  2. Logo abaixo, adicione o seguinte para exibir o fornecedor (vendor) em maiúsculas:
    Liquid / HTML 
    <p>Marca: {{ product.vendor | upcase }}</p>
  3. Adicione um "Fornecedor" a um produto no Admin e visualize a página do produto.

Resposta:

O código a ser adicionado é exatamente o fornecido no enunciado do exercício:

Liquid / HTML 
<p>Marca: {{ product.vendor | upcase }}</p>

Explicação:

  • {{ product.vendor }}: Acede à propriedade vendor do objeto product. Esta propriedade contém o nome do fornecedor/marca que você define na página de edição do produto no admin do Shopify.
  • | upcase: Este é um filtro Liquid que converte todo o texto da string resultante de product.vendor para letras MAIÚSCULAS.
  • O resultado é envolvido por tags de parágrafo <p>Marca: ... </p> para exibição.

Após adicionar este código ao ficheiro templates/product.liquid (geralmente abaixo da linha que exibe {{ product.title }}) e definir um valor para o campo "Fornecedor" (Vendor) na página de edição de um produto no seu admin Shopify, você deverá ver o nome do fornecedor em maiúsculas na página daquele produto na sua loja.

Exercício Prático Módulo 4:
  1. Na página de um produto da sua loja, clique com o botão direito no título do produto e "Inspecionar".
  2. No painel de estilos, tente adicionar uma propriedade CSS para o título, como color: green;.
  3. Vá em Editar código e abra o arquivo CSS principal do seu tema (em assets/).
  4. Adicione uma regra CSS para mudar a cor de fundo de todos os botões (button ou uma classe comum de botão do seu tema) para uma cor de sua escolha. Salve e visualize.

Resposta (Item 4 - Exemplo de Código CSS):

O nome do arquivo CSS principal e a classe exata dos botões podem variar dependendo do seu tema. No tema Dawn, por exemplo, os botões principais costumam ter a classe .button ou .button--primary.

Exemplo de código para adicionar ao seu arquivo CSS principal (ex: assets/base.css ou assets/theme.css):

CSS 
/* Adicione no final do seu arquivo CSS principal */

/* Exemplo genérico para a tag button */
button {
  background-color: #FFC107; /* Amarelo Âmbar */
  color: #212529; /* Cor de texto escura para contraste */
  border: 1px solid #FFC107; /* Opcional: borda da mesma cor */
}

/* Exemplo mais específico se o seu tema usa uma classe como .button */
.button, 
.button--primary { /* Incluindo uma classe comum do Dawn */
  background-color: #28a745; /* Verde Sucesso */
  color: white;
  border-color: #28a745; /* Para consistência */
}

/* Para garantir que o hover também seja ajustado, se necessário */
.button:hover,
.button--primary:hover {
  background-color: #218838; /* Verde um pouco mais escuro no hover */
  border-color: #1e7e34;
}

Explicação:

  • Primeiro, identifique a tag HTML (button) ou a classe CSS que o seu tema usa para os botões principais (use a ferramenta "Inspecionar" do navegador para isso).
  • Adicione a nova regra CSS no final do seu arquivo CSS para garantir que ela tenha precedência ou modifique a regra existente se preferir.
  • background-color: #SUA_COR_AQUI; define a cor de fundo.
  • É uma boa prática também definir a color do texto para garantir bom contraste e, possivelmente, a border-color.

Após salvar o arquivo CSS, atualize a página da sua loja para ver a mudança na cor dos botões.

Exercício Prático Módulo 5:
  1. Na templates/product.liquid, adicione um botão: <button id="showDescriptionButton">Mostrar Descrição</button>.
  2. Adicione um <div id="productDescriptionOutput" style="display:none;"></div>.
  3. No seu arquivo JS principal (em assets/):
    • Ao carregar o DOM, encontre o botão e o div.
    • Adicione um evento de clique ao botão.
    • Quando clicado, pegue a descrição do produto (você pode precisar passá-la do Liquid para o JS, talvez usando um atributo de dados no HTML ou um script inline que defina uma variável JS com {{ product.description | json }}) e coloque-a dentro do div#productDescriptionOutput, tornando-o visível. Altere o texto do botão para "Ocultar Descrição".
    • Implemente a funcionalidade de ocultar também.

Resposta:

1. Adicionar HTML em templates/product.liquid:

Coloque isto onde você quer que o botão e a descrição apareçam (ex: abaixo das informações principais do produto):

Liquid / HTML 
<button id="showDescriptionButton" class="button button--primary">Mostrar Descrição Completa</button>
<div id="productDescriptionOutput" style="display:none; margin-top: 15px; padding: 10px; border: 1px dashed #ccc;">
    <!-- A descrição será inserida aqui -->
</div>

<!-- Script para passar a descrição do Liquid para o JavaScript de forma segura -->
<script type="text/template" id="productDescriptionData">
    {{ product.description | json }}
</script>

Nota: Usamos <script type="text/template"> para guardar os dados JSON. É uma forma comum de passar dados do servidor para o JS sem executá-los diretamente como script.

2. Adicionar JavaScript ao seu arquivo JS principal (ex: assets/theme.js ou assets/global.js):

JavaScript 
document.addEventListener('DOMContentLoaded', function() {
  const descriptionButton = document.getElementById('showDescriptionButton');
  const descriptionOutput = document.getElementById('productDescriptionOutput');
  const descriptionDataElement = document.getElementById('productDescriptionData');

  if (descriptionButton && descriptionOutput && descriptionDataElement) {
    // Parseia o JSON da descrição do produto
    // O .trim() é importante caso haja espaços em branco em volta do JSON
    const productDescription = JSON.parse(descriptionDataElement.innerHTML.trim());

    descriptionButton.addEventListener('click', function() {
      if (descriptionOutput.style.display === 'none') {
        descriptionOutput.innerHTML = productDescription; // Insere a descrição HTML
        descriptionOutput.style.display = 'block';
        descriptionButton.textContent = 'Ocultar Descrição';
      } else {
        descriptionOutput.style.display = 'none';
        descriptionOutput.innerHTML = ''; // Limpa para não manter conteúdo desnecessário
        descriptionButton.textContent = 'Mostrar Descrição Completa';
      }
    });
  }
});

Explicação do JavaScript:

  • Espera o DOM carregar completamente.
  • Obtém referências para o botão, o div de saída e o elemento script que contém os dados da descrição.
  • Lê o conteúdo do <script type="text/template"> e usa JSON.parse() para converter a string JSON (que contém a descrição do produto) de volta para HTML utilizável.
  • Adiciona um "escutador de evento" de clique ao botão.
  • Quando o botão é clicado:
    • Verifica se o descriptionOutput está oculto (display === 'none').
    • Se estiver oculto: define o innerHTML do div com a descrição, muda o display para 'block' (para mostrá-lo) e atualiza o texto do botão.
    • Se estiver visível: muda o display para 'none', limpa o innerHTML e restaura o texto do botão.

Lembre-se de que o seu arquivo JS principal deve estar incluído no layout/theme.liquid, geralmente antes do fechamento da tag </body>.

Exercício Prático Módulo 6:
  1. No Editor de Temas, explore as seções da página inicial. Tente adicionar uma seção "Depoimentos".
  2. Mude as cores secundárias do seu tema em "Configurações do tema" > "Cores".
  3. Para uma edição de código: Duplique seu tema. No editor de código, abra sections/footer.liquid. Encontre o texto de direitos autorais e adicione "Todos os direitos reservados." ao lado. Salve e visualize.

Resposta (Item 3 - Exemplo de Código):

Os itens 1 e 2 são ações a serem realizadas diretamente no Editor de Temas do Shopify.

Para o item 3, a modificação no ficheiro sections/footer.liquid (ou similar, o nome pode variar um pouco dependendo do tema) seria algo assim:

Antes (exemplo comum):

Liquid / HTML 
<div class="footer-copyright">
  &copy; {{ "now" | date: "%Y" }}, {{ shop.name | link_to: routes.root_url }}.
  {{ powered_by_link }}
</div>

Depois (com a adição):

Liquid / HTML 
<div class="footer-copyright">
  &copy; {{ "now" | date: "%Y" }}, {{ shop.name | link_to: routes.root_url }}. Todos os direitos reservados.
  {{ powered_by_link }}
</div>

Explicação:

  • Você localiza a linha que normalmente exibe o símbolo de copyright (&copy;), o ano atual ({{ "now" | date: "%Y" }}), e o nome da loja ({{ shop.name }}).
  • Simplesmente adicione o texto "Todos os direitos reservados." após o nome da loja ou onde fizer mais sentido no contexto do seu rodapé.
  • A tag {{ powered_by_link }} (que gera o "Powered by Shopify") é opcional e pode ser removida se desejado, embora alguns temas gratuitos possam ter restrições quanto a isso.

Lembre-se sempre de **duplicar o seu tema** antes de fazer edições diretas no código, para ter um backup seguro.

Para um desenvolvimento mais profissional e eficiente, o Shopify CLI é essencial.

O que é o Shopify CLI?

Shopify CLI (Command Line Interface) é uma ferramenta que permite interagir com a plataforma Shopify diretamente do seu terminal...

Comandos Úteis (Introdução):

  • shopify theme serve: Baixa uma cópia do tema...
  • ... (outros comandos) ...
Exercício Prático Módulo 7:
  1. Instale o Shopify CLI.
  2. No terminal, navegue até uma pasta vazia. Execute shopify login --store SUA_LOJA_DE_DESENVOLVIMENTO.myshopify.com.
  3. Execute shopify theme serve. Siga as instruções para baixar seu tema ativo.
  4. Abra a pasta do tema no seu editor. Faça uma pequena alteração em sections/header.liquid (ex: adicione um comentário HTML). Salve.
  5. Verifique o terminal e visualize sua loja para ver a alteração. Para parar o servidor, pressione Ctrl + C.

Resposta e Verificação:

Este exercício é mais sobre seguir os passos e observar o resultado. Não há um "código de resposta" único, mas sim um processo a ser verificado:

  1. Instalação do Shopify CLI: Verifique se a instalação foi bem-sucedida executando shopify version no seu terminal. Você deverá ver um número de versão.
  2. Login: Após shopify login --store SUA_LOJA.myshopify.com, o navegador deverá abrir para autenticação. Uma vez autenticado, o terminal confirmará.
  3. `shopify theme serve`:
    • O CLI deverá conectar-se à sua loja e perguntar qual tema você deseja desenvolver (ou oferecer para baixar o tema ativo).
    • Após o download (se for a primeira vez) e o início do servidor, você verá URLs no terminal, como:
      Serving...
      Previewing theme in your browser at http://127.0.0.1:9292
      Using theme "Nome do Seu Tema" (ID: 1234567890) on loja-teste.myshopify.com
      Syncing theme files...
  4. Alteração no Código e Visualização:
    • Ao abrir sections/header.liquid e adicionar, por exemplo, <!-- MEU TESTE CLI NO CABEÇALHO --> e salvar o ficheiro:
    • O terminal onde shopify theme serve está a correr deverá mostrar uma mensagem como [development] sections/header.liquid was synced.
    • Ao abrir a URL de preview (ex: http://127.0.0.1:9292) ou a URL da sua loja de desenvolvimento e inspecionar o código fonte do cabeçalho (Ctrl+U ou clique direito > Ver código fonte da página), você deverá encontrar o seu comentário HTML lá.
  5. Parar o Servidor: Pressionar Ctrl + C no terminal onde o serve está a correr deverá parar o processo.

Se todos esses passos ocorreram como descrito, o exercício foi concluído com sucesso!

Resposta:

1. Rodapé com nome da loja e ano atual (em layout/theme.liquid):

Adicione isto antes do fechamento da tag </body> ou onde o seu rodapé estiver localizado:

Liquid / HTML 
<footer style="text-align: center; padding: 20px; border-top: 1px solid #eee; margin-top: 30px;">

2. Listar tags do produto (em templates/product.liquid):

Coloque isto onde você quer que as tags apareçam (ex: abaixo da descrição):

Liquid / HTML 
{% if product.tags.size > 0 %}
  <div class="product-tags" style="margin-top: 20px;">
    <h4>Tags do Produto:</h4>
    <ul style="list-style: none; padding: 0;">
      {% for tag in product.tags %}
        <li style="display: inline-block; background-color: #f0f0f0; padding: 5px 10px; margin-right: 5px; border-radius: 4px;">
          <a href="{{ routes.collections_url }}/all/{{ tag | handleize }}">{{ tag }}</a>
        </li>
      {% endfor %}
    </ul>
  </div>
{% endif %}

Explicação da listagem de tags:

  • {% if product.tags.size > 0 %}: Verifica se o array product.tags contém alguma tag. .size retorna o número de itens.
  • {% for tag in product.tags %}: Itera sobre cada tag no array.
  • {{ tag }}: Exibe o nome da tag.
  • <a href="{{ routes.collections_url }}/all/{{ tag | handleize }}">...</a>: Cria um link para uma página de coleção que (por convenção do Shopify) mostraria todos os produtos com essa tag. routes.collections_url dá a base da URL das coleções, /all/ refere-se a todos os produtos, e {{ tag | handleize }} converte a tag num formato amigável para URL (ex: "Promoção de Verão" -> "promocao-de-verao").

3. Uso de forloop em templates/collection.liquid:

Dentro do seu loop de produtos na página de coleção (ex: {% for product_item in collection.products %}):

Liquid / HTML 
{% assign product_card_classes = "product-card" %}
{% if forloop.first %}
  {% assign product_card_classes = product_card_classes | append: ' is-first-item' %}
{% endif %}
{% if forloop.last %}
  {% assign product_card_classes = product_card_classes | append: ' is-last-item' %}
{% endif %}

<div class="{{ product_card_classes }}">
  <!-- Conteúdo do card do produto -->
  <h4>{{ product_item.title }}</h4>
  <p>Índice: {{ forloop.index }} de {{ forloop.length }}</p> 
  <!-- Fim do conteúdo do card -->
</div>

E no seu CSS (ex: assets/base.css):

CSS 
.product-card.is-first-item {
  border-left: 3px solid blue;
  padding-left: 10px;
}
.product-card.is-last-item {
  border-right: 3px solid green;
  padding-right: 10px;
}

Explicação:

  • O snippet banner-promocional é flexível devido aos parâmetros.
  • Na página de coleção, criamos uma variável product_card_classes. Usamos forloop.first e forloop.last para verificar se é o primeiro ou o último item na iteração atual do loop. Se for, adicionamos (usando o filtro append) uma classe CSS específica.
  • O CSS então aplica estilos distintos a esses itens. Também demonstrei como usar forloop.index (posição atual) e forloop.length (total de itens no loop).

JavaScript (para funcionalidade das abas, em assets/theme.js ou similar):

JavaScript 
document.addEventListener('DOMContentLoaded', function () {
  const tabContainers = document.querySelectorAll('.product-tabs-container');

  tabContainers.forEach(container => {
    const tabButtons = container.querySelectorAll('.tab-buttons [role="tab"]');
    const tabPanels = container.querySelectorAll('.tab-content [role="tabpanel"]');

    if (tabButtons.length === 0) return;

    function activateTab(activeButton) {
      tabButtons.forEach(button => {
        button.setAttribute('aria-selected', 'false');
        button.classList.remove('active');
        const panel = document.getElementById(button.getAttribute('aria-controls'));
        if (panel) {
          panel.style.display = 'none';
          panel.classList.remove('active');
        }
      });

      activeButton.setAttribute('aria-selected', 'true');
      activeButton.classList.add('active');
      const activePanel = document.getElementById(activeButton.getAttribute('aria-controls'));
      if (activePanel) {
        activePanel.style.display = 'block';
        activePanel.classList.add('active');
      }
    }

    tabButtons.forEach(button => {
      button.addEventListener('click', function (event) {
        activateTab(event.currentTarget);
      });
      // Para navegação por teclado (Enter e Space)
      button.addEventListener('keydown', function(event) {
        if (event.key === 'Enter' || event.key === ' ') {
          event.preventDefault();
          activateTab(event.currentTarget);
        }
      });
    });

    // Ativa a primeira aba por defeito
    if (tabButtons.length > 0) {
      activateTab(tabButtons[0]);
    }
  });
});

2. Otimizar um Loop (Exemplo Conceitual):

Suponha que você tenha um loop que precisa de verificar o tipo de cada produto para exibir um ícone, e o tipo é uma string que precisa ser "handleizada" (convertida para formato de URL amigável) para usar como classe CSS.

Não otimizado (dentro do loop):

Liquid 
{% for product_item in collection.products %}
  <div class="icon-{{ product_item.type | handleize }}">...</div>
  <p>{{ product_item.title }}</p>
{% endfor %}

Otimizado (se o handleize for custoso ou se o tipo for usado várias vezes):

Liquid 
{% for product_item in collection.products %}
  {% assign product_type_handle = product_item.type | handleize %}
  <div class="icon-{{ product_type_handle }}">...</div>
  <p>{{ product_item.title }} (Tipo: {{ product_type_handle }})</p>
{% endfor %}

Explicação da Otimização: No exemplo otimizado, o filtro handleize é aplicado uma vez por iteração e o resultado é guardado na variável product_type_handle. Se product_item.type | handleize fosse usado múltiplas vezes dentro do mesmo loop para o mesmo product_item, esta abordagem evitaria aplicar o filtro repetidamente, tornando o código mais limpo e potencialmente um pouco mais eficiente para filtros mais complexos.
